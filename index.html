The Problem

AI AGENTS ARE NOT TRADITIONAL BOTS

Unlike simple web scrapers, they navigate websites the way humans do — clicking, scrolling, and interacting with dynamic content. But getting them treated like humans is another challenge entirely. Residential IPs are a critical piece of that puzzle.

01

DATACENTER IPS GET BLOCKED

Websites detect and block datacenter IP ranges. Your agent gets CAPTCHAs, rate limits, or outright bans before it can do anything useful.

02

BOT DETECTION EVOLVES

Modern anti-bot systems fingerprint connections. IP type is a primary signal — datacenter IPs are flagged instantly, blocking your agent's access.

03

GEO-RESTRICTIONS APPLY

Content varies by region. Without residential IPs in the right geography, your agent sees different — or no — content at all.

The Solution

RESIDENTIAL IPS WITH AGENT-NATIVE TOOLING

Skill, CLI, SDK, API — every interface your agent needs. Standard proxy protocol means zero code changes. Just set a proxy URL and your agent's traffic flows through real residential connections.

Features

EVERYTHING YOUR AGENT NEEDS

Built from the ground up for AI agent workflows. No compromises.

01

HTTP & SOCKS5

Dual-protocol proxy support. HTTP on port 8080, SOCKS5 on port 1080. Same residential node pool behind both.

02

GEOGRAPHIC TARGETING

Route requests through specific countries using ISO 3166-1 codes. Need a Korean IP? Set X-SpaceRouter-Region: KR.

03

ZERO CODE CHANGES

Standard proxy protocol. Just set your proxy URL and authenticate with your API key. Works with any HTTP client.

04

INTELLIGENT ROUTING

Health-scored node selection with weighted preferences. Requests route through the best available node automatically.

05

SECURITY BUILT IN

mTLS between components, SSRF protection, DNS rebinding prevention, rate limiting, and full header stripping before reaching target sites.

06

REQUEST TRACING

Every request gets a unique X-SpaceRouter-Request-Id for end-to-end debugging across the entire routing path.

07

DISTRIBUTED NODES

Home Nodes run on residential machines with automatic NAT traversal via UPnP — no manual router configuration needed.

How It Works

FROM AGENT TO TARGET IN 5 STEPS

The response flows back through the same path. The target only ever sees a residential IP.

Architecture Flow

01

YOUR AI AGENT

HTTP or SOCKS5

Your agent sends a request through the Proxy Gateway using standard proxy protocol.

02

PROXY GATEWAY

Authentication

The Gateway validates your API key and checks rate limits.

03

COORDINATION API

mTLS

Selects the best Home Node based on health, geography, and IP type preference.

04

HOME NODE

Residential IP

Forwards your request from its residential IP to the target website.

05

TARGET WEBSITE

Normal Response

Sees a residential IP, serves content normally. The target only ever sees a residential IP.

Proxy Gateway

Agent-facing entry point. Handles HTTP and SOCKS5 proxy protocols, authentication, rate limiting, and route selection.

Coordination API

Central control plane. Manages the node registry, API keys, IP classification (via IPinfo.io), and health-based routing.

Home Node

Residential exit point. Runs on home machines behind NAT, auto-configures port forwarding via UPnP, and proxies traffic from its residential IP.
