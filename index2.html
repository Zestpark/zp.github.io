    Residential IP as a Service for AI Agents

    Many websites block datacenter IPs. SpaceRouter routes your agent's traffic through real residential connections — transparently, with zero code changes beyond setting a proxy URL.

    Get Started How It Works

    The Problem

    AI Agents are not traditional bots. But they're blocked like them.

    Unlike simple web scrapers, they navigate websites the way humans do — clicking, scrolling, and interacting with dynamic content. But getting them treated like humans is another challenge entirely. Residential IPs are a critical piece of that puzzle. SpaceRouter provides Residential IPs with agent-native tooling (Skill, CLI, SDK, API).

    HTTP & SOCKS5
    Dual-protocol proxy support. HTTP on port 8080, SOCKS5 on port 1080. Same residential node pool behind both.
    Geographic Targeting
    Route requests through specific countries using ISO 3166-1 codes. Need a Korean IP? Set X-SpaceRouter-Region: KR.
    Zero Code Changes
    Standard proxy protocol. Just set your proxy URL and authenticate with your API key. Works with any HTTP client.
    Security Built In
    mTLS between components, SSRF protection, DNS rebinding prevention, rate limiting, and full header stripping before reaching target sites.

    How It Works

    Five steps, one proxy URL

    Your agent sends a standard proxy request — SpaceRouter handles everything else.

    01
    Your Agent
    Sends a request through the Proxy Gateway using standard proxy protocol.
    02
    Proxy Gateway
    Validates your API key and checks rate limits.
    03
    Coordination API
    Selects the best Home Node based on health, geography, and IP type.
    04
    Home Node
    Forwards your request from its residential IP to the target.
    05
    Target Website
    Sees a residential IP, serves content normally. Response flows back through the same path.

    Architecture
    
        
    Three components
    
      
    
      
    Each component has a single responsibility. Together they route your agent's traffic through a real residential IP.
    
    
    
    
    
      
    
        
    Entry point
    
        
    Proxy Gateway
    
        
    Agent-facing entry point. Handles HTTP and SOCKS5 proxy protocols, authentication, rate limiting, and route selection.
    
      
    
      
    
        
    Control plane
    
        
    Coordination API
    
        
    Central control plane. Manages the node registry, API keys, IP classification (via IPinfo.io), and health-based routing.
    
      
    
      
    
        
    Exit point
    
        
    Home Node
    
        
    Residential exit point. Runs on home machines behind NAT, auto-configures port forwarding via UPnP, and proxies traffic from its residential IP.
    
      
    
    
    
  
    

    



    
  
    
    
    
      
    
        
    Phase 1.1 — Live now
    
        
    Get started
    in two steps
    
        
    Create an API key, point your proxy URL, and your next request routes through a real residential IP. No infrastructure to manage, no code to change.
    
      
    
      
    
        
    
          
    
            
            
            
            bash
          
    
          
    # Step 1 — Create an API key
curl -X POST https://coordination.spacerouter.org/api-keys \
  -H "Content-Type: application/json" \
  -d '{"name": "my-agent"}'

# Step 2 — Use the proxy
curl -x http://YOUR_API_KEY@gateway.spacerouter.org:8080 \
  https://httpbin.org/ip

# That's it. Your request is now routed through a residential IP.
